# CinemaFlo — In-Cinema Commerce Platform

**ApexFlo MTS-1 Engineering Assignment**

CinemaFlo is a scalable in-cinema food ordering platform that enables patrons to browse concession menus, place orders, apply offers, and track order status while guaranteeing zero inventory overselling.

---

# Quick Start

## Option 1: Run with Docker (Recommended)

```bash
docker-compose up
```

## Option 2: Run Locally

### Install Dependencies

```bash
pip install -r requirements.txt
```

### Start the Application

```bash
python -m uvicorn api.main:app --host 0.0.0.0 --port 8000 --reload
```

# Oversell Prevention Strategy

The most critical requirement of CinemaFlo is ensuring inventory can never be oversold.

Stock reservation is performed using an atomic SQL update:

```sql
UPDATE stock
SET quantity = quantity - ?, last_updated = datetime('now')
WHERE item_id = ? AND quantity >= ?
```

### Why It Works

* SQLite serializes write operations.
* The `WHERE quantity >= ?` condition guarantees sufficient inventory.
* Failed updates return `rowcount = 0`.
* Database constraints prevent negative inventory.

Result:

```text
50 concurrent buyers
1 remaining item

→ 1 successful purchase
→ 49 rejected requests
→ 0 oversells
```
